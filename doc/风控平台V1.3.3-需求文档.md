# 风控平台 V1.3.3 需求文档

## 1. 文档目标

本版本用于补充“多维画像查询”页面中的**多场景排序与推荐查看逻辑**，重点解决以下问题：

1. 同一查询对象在近 7 天内可能同时命中登录、注册、支付等多个异常场景；
2. 页面需要在首屏给出“优先看哪个场景”的引导；
3. 页面结构需要稳定，不能因为排序结果不同而让 Tab 顺序频繁变化；
4. 排序结果需要可解释，便于产品、运营和研发统一理解。

## 2. 适用范围

| 项目 | 说明 |
| --- | --- |
| 页面 | 用户画像中心 > 多维画像查询 |
| 查询对象 | 账号名、SDKID、设备ID、IP地址 |
| 场景类型 | 登录异常、注册异常、支付异常 |
| 时间口径 | 近 7 天异常 + 近 30 天历史基线 |

## 3. 设计原则

| 原则 | 说明 |
| --- | --- |
| 结构稳定 | 场景 Tab 顺序固定，不因不同账号的结果而动态换位 |
| 引导明确 | 在固定顺序前提下，对最值得优先查看的场景增加“推荐查看”标记 |
| 结果可解释 | 推荐场景必须能说清楚推荐依据，不能只给黑盒结果 |
| 能力诚实 | 当前仅支持部分场景已有模型能力，不伪装成全异常统一判定器 |
| 可扩展 | 后续新增场景或模型时，能沿用同一排序框架 |

## 4. 页面结构约束

### 4.1 场景总览条

首屏顶部不再使用普通“总览摘要”，改为“场景总览条”，承担跨场景导航功能。

建议包含以下信息：

| 字段 | 说明 |
| --- | --- |
| 核实状态 | 近 7 天存在异常 / 近 7 天暂未发现明显异常 |
| 场景分布 | 登录异常 X、注册异常 Y、支付异常 Z |
| 首看建议 | 建议优先查看：登录异常 / 注册异常 / 支付异常 |
| 推荐依据 | 根据最高风险事件、异常次数、最近发生时间综合排序 |

### 4.2 场景 Tab

| 规则 | 说明 |
| --- | --- |
| 顺序 | 固定顺序展示：登录异常、注册异常、支付异常 |
| 推荐标记 | 对排序结果第一的场景 Tab 增加“推荐查看”标记 |
| 默认激活 | 默认进入排序结果第一的场景 |
| 未命中状态 | 场景无异常时仍保留 Tab，但展示“暂未发现明显异常”或弱化状态 |

## 5. 场景排序逻辑

### 5.1 目标

在多个场景同时存在异常时，选出**最值得用户优先查看**的场景，用于：

1. 场景总览条中的“首看建议”；
2. 场景 Tab 上的“推荐查看”标记；
3. 页面首次进入时的默认激活场景。

### 5.2 排序维度

排序不直接使用“单一总分”，而使用以下三个维度组合判断：

| 维度 | 含义 | 作用 |
| --- | --- | --- |
| 最高风险事件强度 | 近 7 天内该场景最危险的一次事件 | 判断“哪类问题最严重” |
| 近期异常事件次数 | 近 7 天内该场景的异常事件数量 | 判断“问题是否持续发生” |
| 最近异常发生时间 | 该场景最近一次异常距离当前的时间 | 判断“问题是否仍在持续或刚刚发生” |

### 5.3 推荐排序规则

建议按以下顺序比较：

| 排序层级 | 规则 |
| --- | --- |
| 1 | 先比较近 7 天内是否存在高风险事件 |
| 2 | 若多个场景均存在高风险事件，比较最高风险事件强度 |
| 3 | 若最高风险事件强度接近，比较近 7 天异常事件次数 |
| 4 | 若异常次数接近，比较最近一次异常发生时间，越近优先级越高 |
| 5 | 若以上维度仍无法区分，按固定兜底优先级排序 |

### 5.4 固定兜底优先级

当前建议兜底顺序如下：

1. 登录异常
2. 注册异常
3. 支付异常

说明：

1. 当前首期重点服务盗号核实，登录异常场景最成熟；
2. 注册、支付场景后续可补强，不应在首期夺走默认核实主线；
3. 后续如某项目主打防刷号或支付风险，可支持项目级配置调整兜底顺序。

## 6. 推荐结果表达方式

### 6.1 场景总览条示例

```text
近 7 天存在异常，异常主要集中在登录场景。
登录异常 3，注册异常 1，支付异常 0。
建议优先查看：登录异常。
推荐依据：近 7 天最高风险事件出现在登录场景，且最近一次异常发生于 2 小时前。
```

### 6.2 Tab 展示示例

| Tab | 展示方式 |
| --- | --- |
| 登录异常 | 登录异常 `推荐查看` |
| 注册异常 | 注册异常 |
| 支付异常 | 支付异常 |

## 7. 口径说明

| 项目 | 说明 |
| --- | --- |
| “异常事件” | 指命中该场景风险识别规则或策略区间的事件 |
| “最高风险事件强度” | 可使用该场景近 7 天内单次事件最高风险分或最严处置区间表达 |
| “近期异常事件次数” | 近 7 天内被标记为异常的事件次数，不等于全部行为次数 |
| “默认激活场景” | 指页面首次展示结果时默认打开的 Tab，不影响用户后续手动切换 |

## 8. 当前边界

| 边界 | 说明 |
| --- | --- |
| 不输出统一总分 | 当前不把多场景结果压缩成一个账号级统一风险分 |
| 不直接做跨场景定性 | 当前不直接给出“更像盗号 / 更像刷号 / 更像代充”的全自动结论 |
| 推荐不等于裁决 | “推荐查看”仅表示优先查看顺序，不代表最终风险结论 |

## 9. 后续可扩展方向

| 方向 | 说明 |
| --- | --- |
| 项目级排序配置 | 不同项目可配置兜底优先级 |
| 场景级说明文案 | 每个场景可展示“推荐原因”明细 |
| 场景摘要卡 | Tab 内增加该场景专属概览卡 |
| 倾向判断能力 | 待各场景模型成熟后，再补“更像盗号 / 更像刷号 / 更像代充”等业务倾向判断 |
