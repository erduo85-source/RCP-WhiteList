# 多维画像查询低保真结构稿说明

## 1. 文档定位

| 项目 | 内容 |
| --- | --- |
| 目标 | 将正式页面方案转成可评审的低保真结构稿说明 |
| 用途 | 用于确认模块顺序、信息层级、状态设计、关键交互 |
| 边界 | 本文档不定义视觉风格，不定义颜色、插画、图标风格、字体细节 |
| 设计基线 | 以 `doc/多维画像查询-页面方案.md` 的“当前最终口径快照”为准 |

## 2. 页面容器

| 项目 | 说明 |
| --- | --- |
| 容器形态 | 大弹窗 |
| 打开方式 | 可由其他页面快速调用 |
| 关闭方式 | 右上角关闭按钮 |
| 内部交互约束 | 不再叠加二次弹窗或抽屉，优先使用页内展开、收起、切换 |

## 3. 整体结构

从上到下的低保真结构如下：

| 顺序 | 区域 | 目标 |
| --- | --- | --- |
| 1 | 弹窗标题区 | 标识当前查询对象和组件名称 |
| 2 | 基础信息区域 | 在不打断主线的前提下，保留对象背景信息 |
| 3 | 场景 Tab | 切换登录 / 注册 / 支付 |
| 4 | 登录分布模块 | 展示标题、一行摘要和风险分布图 |
| 5 | 关联分析区 | 用于发现更深层关系问题 |
| 6 | 登录日志区 | 展示风险 / 全量记录和完整明细 |

## 8. 三种关键状态

### 8.1 默认风险态

适用于“近7天存在明显异常”的常见核实场景。

#### 结构表现

1. `登录分布` 模块头部展示一行风险摘要
2. 分布图中的风险节点明显可见
3. 日志区默认展示风险记录
4. 关联分析区与登录日志标题都能在首屏看到

### 8.2 无明显异常态

适用于“未发现明显异常”的核实结果。

#### 结构表现

1. `登录分布` 模块头部展示低风险或空安全值摘要
2. 分布图可弱化展示
3. 日志区仍可切到全部记录核查
4. 关联分析区仍保留，用于深挖历史关系

### 8.3 多场景并存态

适用于登录、注册、支付同时存在异常的情况。

#### 结构表现

1. 页签通过红点或数字展示各场景风险状态
2. 默认进入 `登录`
3. 当前场景直接进入自己的分布、关联分析、日志区

## 10. 评审重点

在低保真评审阶段，优先确认以下问题：

| 重点 | 检查问题 |
| --- | --- |
| 结构顺序 | 是否符合“先判断全局，再下钻场景，再核查明细”的路径 |
| 模块边界 | 概览卡、核实依据区、时间线、日志区是否有重复 |
| 信息密度 | 是否存在默认态信息过重、挤占主视区的问题 |
| 状态设计 | 默认风险态、无明显异常态、多场景并存态是否都说得通 |
| 图谱价值 | 关联图谱是否真的有助于进一步发现问题，而不只是可视化装饰 |
