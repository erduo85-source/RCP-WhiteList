# 风控平台V1.3.1-需求文档

# 需求背景

:::
**问题**：黑产攻击手段多样（撞库、垃圾注册）、风险场景复杂（异地/新设备登录）、处置方式粗放（一刀切拦截），导致安全与体验难以平衡。
:::
:::
**目标**：构建端云协同的智能风控体系，形成可解释、可配置、可追踪的闭环风控平台。

1. 梯度化处置  

   实现风险梯度打分，降低对正常用户的影响。

2. 策略热生效  

   支持后台配置、快速启停，策略开启后 5 分钟内完成线上防护生效。

3. 场景全覆盖  

   覆盖登录、注册、支付三大核心场景，打通风险识别、行为分析、策略处置和结果回溯的闭环防护。

4. 数据驱动运营  

   通过安全概览、风控日志、风险行为分析帮助业务方快速识别风险漏洞，并推动风控策略逐步启用。
:::

3.  **检测开关：**项
    
4.  **统计指标：**展示当前风险行为判断的核心计算指标，支持配置计算规则的阈值
    
5.  **得分规则：**包括提示文案+配置表
    
    *   提示文案：由服务端传入（不同风险行为的提示文案可能有差异）
        
    *   表格表头：最小值、统计指标、最大值、风险分、操作
        
    *   最小值：首行的最小值固定为0；其他行的最小值不支持配置，其数值自动同步为上一行的最大值
        
    *   统计指标：由服务端传入（不同风险行为的统计指标可能有差异）
        
    *   最大值：
        
        *   操作说明：最后一行的最大值固定为+∞，不支持配置；其它行的最大值均支持配置
            
        *   校验规则0：**必填**；校验不通过时，当前行的最大值输入框标红，并且toast提示“请输入最大值”
            
        *   校验规则1：**当前行的最大值 需要＞当前行的最小值**；校验不通过时，当前行的最大值输入框标红，并且toast提示“最大值不能小于最小值，请重新输入”
            
        *   校验规则2：**当前行的最大值 需要＜ 下一行的最大值**；校验不通过时，当前行的最大值和下一行的最大值 两个输入框标红，并且toast提示“最大值不能超过更大分数区间的最大值，请重新输入”
            
        *   交互效果：输入框清空时，下一行的最小值同步清空；输入框校验通过后，下一行的最小值同步更新
            
    *   风险分：
        

*   填写校验：必填；未输入时，输入框标红，并且toast提示“请输入风险分”
    
    *   数据校验：仅允许输入数字，屏蔽其他类型字符；支持输入0~100的数字，校验不通过时输入框标红并toast提示“请输入0~100的风险分”
        
*   删除：
    
    *   操作说明：第一行和最后一行不支持删除，按钮置灰；其他行支持删除
        
    *   交互效果：**删除后，自动合并分数区间**；
        
    *   举例说明： 第一行为0~30 + 第二行为30~70 + 第三行为70~+∞  → 删除第二行后，第三行变为30~+∞
        
*   添加：
    
    *   操作说明：每一行均支持添加操作
        
    *   交互效果：点击添加后，在当前行下方新增一行，并同步最小值，并清空新增行下方的最小值；
        
    *   举例说明：第二行A为30~40，第三行B为40~50 → 点击第二行的添加，增加行C到第三行，展示为40~\[\]；行B变为第四行，数值变为\[\]~50
        

1.  **保存按钮：**点击后校验页面所有输入项，验证通过后关闭弹窗，并展示toast提示“风险行为规则保存成功”
