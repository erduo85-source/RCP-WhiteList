## 3. 实现目标

本次实现目标是补全“用户画像查询页面”的完整前端能力，形成可演示、可点击、可评审、可继续细化的原型级实现。

### 3.1 入口页目标

入口页负责：

1. 在“画像中心 > 用户画像查询”菜单下提供真实落地页面
2. 支持按账号、设备、IP 三种维度发起查询
3. 展示历史查询入口与近期高风险对象入口
4. 通过统一交互进入查询结果弹窗

### 3.2 查询弹窗目标

查询结果弹窗负责：

1. 承载当前对象的异常核实流程
2. 以登录场景为首期完整展示内容
3. 覆盖基础信息、轻量概览条、场景页签、指标卡、登录分布、关联分析、登录日志
4. 提供风险记录 / 全部记录切换与分页明细

## 4. 实现边界

以下口径必须在实现阶段严格执行，避免开发中重新做产品决策。

| 主题 | 实现口径 |
| --- | --- |
| 数据层 | 全量 mock 闭环，不接接口 |

| 历史查询 | mock 数据，最多 8 条，不实现去重 |
| 查询触发 | 搜索提交后直接打开结果大弹窗 |

| 列表分页 | 固定 10 条 / 页 |

| 搜索校验 | 首版仅校验非空 |

| 视觉验收 | 必须对照两张设计图做浏览器级自检，不以“无报错”代替验收 |

## 6. 组件清单与 antd 选型约束

实现前先完成组件 inventory，再开始编码。以下清单为当前默认选型，除非实现时发现官方组件能力明确不足，否则不改。

| UI 区域 | 需求 | Candidate antd components | Recommended choice | Needs rewrite? | Rewrite scope |
| --- | --- | --- | --- | --- | --- |
| 首页页眉 | 标题 + 一句说明 | `Typography`, `Card` | `Typography + Card/容器` | 否 | 仅做布局与间距定制 |
| 首页账号查询 | 紧凑选择器 + 输入框 + 查询按钮 | `Space.Compact + Select + Input + Button`, `Input.Group` | `Space.Compact + Select + Input + Button` | 否 | 只调宽度、间距、对齐 |
| 首页设备/IP查询 | 单输入框 + 主动作 | `Space.Compact + Input + Button`, `Input + Button` | `Space.Compact + Input + Button` | 否 | 保持结构统一 |
| 历史查询记录 | 轻量胶囊入口 | `Tag`, `Button`, `Space` | `Tag + Space` 或弱按钮 | 轻定制 | 只做可点击胶囊态，不自造复杂组件 |
| 高风险列表 | 混合对象列表 + 分页 + 操作 | `Table`, `Pagination`, `Button` | `Table` 自带分页 + `Button` | 否 | 列渲染与图标映射 |
| 结果弹窗容器 | 覆盖首页的大弹窗 | `Modal` | `Modal` | 否 | 调整宽高、遮罩、页内滚动 |
| 基础信息收起栏 | 一行三列 + 可收起 | `Collapse`, `Card`, `Descriptions` | 轻容器 + `Descriptions`/自定义栅格 | 轻定制 | 不做复杂折叠动画 |
| 轻量概览条 | 状态、计数、优先查看提示 | `Alert`, `Tag`, `Badge`, `Space` | 轻定制信息条 | 是 | 用标准容器拼装，不用重告警组件硬套 |
| 场景页签 | 登录 / 注册 / 支付切换 | `Tabs`, `Segmented` | `Tabs` | 否 | 只调标签态和徽标 |
| 指标卡 | 3 张数值卡 | `Card`, `Statistic` | `Card + Statistic` | 否 | 调整图标和数字层级 |
| 登录分布 | 时间序列分布图 | 纯 antd 无官方图表组件 | 轻量自绘容器 | 是 | 首版允许用 DOM/CSS/轻数据映射实现，不引入新图表库 |
| 关联分析 | 中心对象 + 关联节点 | 纯 antd 无官方图谱组件 | 静态布局化容器 | 是 | 用标准容器、图标、连线样式拼装 |
| 登录日志 | 风险/全部记录切换 + 表格 + 分页 | `Segmented`, `Tabs`, `Table`, `Button` | `Segmented + Table` | 否 | 保持表头、分页、空态一致 |
| 占位场景 | 注册 / 支付未完成态 | `Empty`, `Card` | `Empty + Card` | 否 | 文案和留白定制 |

### 6.1 组件使用规则

| 主题 | 规则 |
| --- | --- |
| 官方 API | 只使用 antd 官方文档中有的 props / events / 组件名 |
| DOM 依赖 | 不依赖内部 DOM 结构，不写广义 `.ant-*` 全局覆盖 |
| 表格 | 必须提供稳定 `rowKey`；分页、排序、切换口径统一 |
| 表单 | 账号查询优先使用 `Space.Compact` 维持紧凑结构 |
| 弹窗 | 统一使用 `Modal`，不切换成 Drawer 或独立页 |
| 切换控件 | 页签用 `Tabs`；风险记录 / 全部记录切换优先 `Segmented` |
| 样式覆盖 | 优先组件 props、tokens、`classNames` / `styles`，CSS 兜底 |

## 11. 风险与控制建议

| 风险 | 表现 | 控制建议 |
| --- | --- | --- |
| 继续堆在 `App.tsx` | 画像查询实现难维护，后续难扩展 | 必须拆出 `features/profile-query` |
| 首页过重 | 抢了结果弹窗职责 | 严格执行首页轻量边界 |
| 弹窗场景贪多 | 注册 / 支付被临时拉成完整模块 | 锁定“登录完整、注册/支付占位” |
| mock 数据散落 | 页面间口径不一致 | 统一收敛到 `shared/mock.ts` |
| 只过 build 不验视觉 | 页面可编译但不可评审 | 必做浏览器级视觉自检 |
| 组件选型失控 | 同一页面混入过多自造控件或样式耦合 | 先做组件 inventory，优先 antd 标准组件，样式遵循 token-first |
