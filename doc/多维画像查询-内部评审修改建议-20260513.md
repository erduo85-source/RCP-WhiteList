# 多维画像查询-内部评审修改建议

| 项目 | 内容 |
| --- | --- |
| 生成时间 | 2026-05-13 20:14:46 |
| 来源 | 内部评审同步意见 |
| 适用范围 | 多维画像查询基础页、查询弹窗 |
| 状态 | 待纳入后续页面方案与实现任务 |

## 一、评审结论

本轮评审的核心方向是收敛功能范围，强化“搜索即入口、弹窗即结论”的体验：

| 序号 | 模块 | 评审意见 | 结论 |
| --- | --- | --- | --- |
| 1 | 基础页面 | 仿照主流搜索页，例如百度，只提供搜索能力 | 基础页进一步收敛为单一搜索入口，不承载额外信息模块 |
| 2 | 查询弹窗 | 需要补充标签条，快速传达账号最近有哪些风险项、有哪些生效标签 | 弹窗增加标签条，优先展示风险标签与生效标签 |
| 3 | 查询弹窗 | 分布图是否有必要 | 暂不默认保留，建议作为待确认项处理 |

## 二、修改建议

### 1. 基础页面调整为纯搜索页

| 项目 | 建议 |
| --- | --- |
| 页面定位 | 基础页只承担“输入查询条件并发起搜索”的职责 |
| 页面结构 | 保留品牌/标题、搜索框、搜索按钮、必要的查询提示 |
| 删除建议 | 移除会分散注意力的说明区、统计区、扩展入口区、预览型内容模块 |
| 体验目标 | 用户进入页面后立即理解“这里只做查询” |

补充说明：
- 可参考百度、主流站内搜索页的结构，页面中心聚焦一个搜索动作。
- 如果需要保留帮助信息，建议降级为搜索框下方一行轻提示，不单独占用版面。

### 2. 查询弹窗补充标签条

| 项目 | 建议 |
| --- | --- |
| 位置 | 放在账号基础信息下方、详细内容上方，作为首屏关键信息层 |
| 信息类型 | 最近风险项、生效标签 |
| 展示形式 | 使用可快速扫读的标签条/Tag 组，按风险优先级排序 |
| 展示原则 | 先展示“当前最需要关注的信息”，避免用户先看长段落或复杂图表 |

建议标签条至少覆盖以下两类信息：

| 标签类型 | 示例 |
| --- | --- |
| 风险标签 | `近7日异地登录`、`高频申诉`、`命中设备风险` |
| 生效标签 | `账号白名单生效中`、`人工关注中`、`高价值账号` |

补充说明：
- 风险标签建议突出风险属性，可使用更高识别度的颜色体系。
- 生效标签建议保持中性或正向提示，避免与风险标签混淆。
- 若标签数量过多，首行展示核心标签，其余折叠或收起处理。

### 3. 查询弹窗中的分布图改为待确认项

| 判断项 | 建议 |
| --- | --- |
| 是否默认保留 | 不建议当前版本默认保留 |
| 原因 | 当前目标是快速完成账号判断，分布图的信息密度高，但未证明是首要决策信息 |
| 替代方案 | 优先展示标签条、关键摘要、重点风险说明 |
| 处理建议 | 在后续评审中确认：分布图是“必须信息”还是“增强信息” |

建议判断标准：

| 判断问题 | 若答案为“是”则可考虑保留 |
| --- | --- |
| 分布图是否直接影响用户判断账号风险结论 | 是 |
| 分布图是否能替代一段较长文字说明 | 是 |
| 分布图是否在首屏内即可被快速理解 | 是 |

若以上条件不成立，建议本期去掉分布图，避免弹窗信息过载。

## 三、建议同步到现有方案文档的方向

| 文档 | 建议同步内容 |
| --- | --- |
| 页面方案 | 将基础页定义调整为“纯搜索页”，删除非必要模块说明 |
| 低保真结构稿说明 | 调整首页结构描述，突出单搜索入口布局 |
| 可执行方案 | 在查询弹窗部分新增“标签条”模块说明与优先级 |
| 实现任务 | 增加标签条实现项；将分布图改为待确认或可选项 |

## 四、落地优先级

| 优先级 | 事项 | 说明 |
| --- | --- | --- |
| P0 | 基础页收敛为纯搜索页 | 影响整体产品定位与首屏认知 |
| P0 | 查询弹窗补充标签条 | 影响查询结果的核心表达效率 |
| P1 | 评估是否保留分布图 | 属于信息增强项，应在核心体验明确后决定 |
