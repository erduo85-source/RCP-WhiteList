# 风控平台V1.3.2-需求文档（修订版）

# 需求背景

:::
**问题**：黑产攻击手段多样（撞库、垃圾注册）、风险场景复杂（异地/新设备登录）、处置方式粗放（一刀切拦截），导致安全与体验难以平衡。
:::
:::
**目标**：构建端云协同的智能风控体系，形成可解释、可配置、可追踪的闭环风控平台。

1. 梯度化处置  
   实现风险梯度打分，降低对正常用户的影响。

2. 策略热生效  
   支持后台配置、快速启停，策略开启后 5 分钟内完成线上防护生效。

3. 场景全覆盖  
   覆盖登录、注册、支付三大核心场景，打通风险识别、行为分析、策略处置和结果回溯的闭环防护。

4. 数据驱动运营  
   通过安全概览、风控日志、风险行为分析帮助业务方快速识别风险漏洞，并推动风控策略逐步启用。
:::

**页面说明**

![image](https://docs-body.xiaoshanim.com/res/VXgNlD85J8ozj8R7/img/55bbcb96-6506-4212-b323-7032a06e1ec7.png?Expires=1778318540&OSSAccessKeyId=file-s3-gateway-ak&Signature=ZPhpLvurvNuzQYdHu0TB33JVLhs%3D)

1. **页面权限：**当且仅当登录账号同时具有“当前页面权限”和“项目权限”时，可见该页面导航
2. **页面入口：**游戏工具 - 运营工具 - 风控管理 - 白名单管理
3. **页面结构：**页面由页眉区域、筛选区域、结果列表区域组成
4. **项目归属：**白名单在当前选中项目维度下生效，不同项目间互不影响；页面右上角提供项目切换器，示例项目包括项目1、项目2、项目3
5. **页眉标题：**
   * 主标题：白名单管理
   * 副标题：可针对账号、设备或IP设置风控白名单，加入白名单的对象不再触发风控
6. **页签切换：**支持切换账号/设备/IP，默认选中账号
7. **筛选-生效对象：**支持按生效对象（账号/设备/IP）进行筛选
   * 标题：根据页签选中项，切换后同步更新，展示为账号/设备/IP
   * 占位符：根据页签选中项同步更新，展示为“请输入账号名或SDKID”/“请输入设备ID”/“请输入IP地址”
   * 边界值：最大允许输入 200 字符，超出时截断输入
   * 默认值：默认无输入，展示占位符
   * 匹配规则：子串匹配/包含匹配
8. **筛选-生效状态：**支持按生效状态进行筛选
   * 标题：固定为生效状态
   * 交互效果：点击后展开/收起单选下拉菜单
   * 枚举值：生效中 / 已失效 / 未开启 / 全部
   * 默认值：默认为全部
   * 选中态：选中后展示已选中的选项
9. **筛选-查询：**点击后提交查询，列表区域展示 loading
10. **筛选-重置：**点击后清空所有筛选项，但不立即触发查询
11. **列表：**
    * 标题：根据页签选中项，切换后同步更新，展示为账号白名单/设备白名单/IP白名单
    * 默认状态：默认展示全部名单数据
    * 排序规则：默认按最近操作时间降序排序
    * 分页：列表底部展示分页器和总数文案，格式为“第 x-y 条 / 共 n 条”
    * 异常状态：包括查询为空状态、查询失败状态、默认状态
    * 字段-生效对象
      * 页签选择账号时：展示 SDKID 和账号名两列
      * 页签选择设备时：展示设备ID
      * 页签选择IP时：展示IP地址
      * 交互效果：支持点击 icon 快速复制到剪切板
      * 展示规则：数据超出列宽时，使用“...”表示
    * 字段-账号名
      * 只展示设置过的账号名，不展示 `.dsp` 等自动生成且不对外的账号名
      * 若导入时仅填写 SDKID，后续账号绑定了账号名后，后台数据需要补充账号名
    * 字段-生效状态
      * 枚举值：生效中、已失效、未开启
    * 字段-生效时间
      * 数据格式：yyyy-mm-dd hh:mm:ss
    * 字段-失效时间
      * 数据格式：yyyy-mm-dd hh:mm:ss
    * 字段-添加原因
      * 展示规则：数据超出列宽时，使用“...”表示
      * 数据来源：服务端传入，首次添加白名单时填入
    * 字段-最近操作人
      * 数据来源：服务端传入
      * 数据更新：新增、编辑、导入覆盖更新时间后同步更新
      * 数据格式：形如 wangjian02
    * 字段-最近操作时间
      * 数据来源：服务端传入
      * 数据更新：新增、编辑、导入覆盖更新时间后同步更新
      * 数据格式：yyyy-mm-dd hh:mm:ss
    * 操作-编辑：点击后展开编辑弹窗，交互细节见下文章节“手动添加”
    * 操作-删除：点击后触发二次确认弹窗，交互细节见下文章节“手动添加”
12. 新增白名单：点击后展示配置弹窗，交互细节见下文章节“手动添加”
13. 批量导入：点击后展开右侧配置抽屉，交互细节见下文章节“批量导入”

**手动添加**

1. **流程-新增白名单：**

![image](https://docs-body.xiaoshanim.com/res/VXgNlD85J8ozj8R7/img/9940ce4d-fb90-4350-8c0b-a9b41f04bc74.png?Expires=1778318540&OSSAccessKeyId=file-s3-gateway-ak&Signature=Rbwf6XylmHBG0%2FXsxjxzLWZfbmM%3D)

* 弹窗类型：包括三种，同步页签选中状态，分为账号白名单、设备白名单、IP白名单
* 弹窗标题：新增账号白名单 / 新增设备白名单 / 新增IP白名单
* 新增账号白名单：
  * 账号信息：使用“账号类型 + 账号信息”的紧凑组合输入
  * 账号类型选择：下拉单选；默认选中账号名；枚举值包括账号名、SDKID；必填
  * 账号信息输入：占位符联动账号类型选择，分别展示为“请输入账号名”/“请输入SDKID”；最大允许输入 100 字符；必填
  * 生效时间：必填，开始时间和结束时间均必填；占位符“请选择生效时间”和“请选择失效时间”；最大允许选择时长不超过 1 年；不允许选择过去时间
  * 添加原因：必填，最大允许输入 200 字符；占位符“请输入添加白名单的原因”
* 新增设备白名单：
  * 设备信息输入：占位符展示为“请输入设备ID”；最大允许输入 100 字符；必填
  * 生效时间：必填，开始时间和结束时间均必填；占位符“请选择生效时间”和“请选择失效时间”；最大允许选择时长不超过 1 年；不允许选择过去时间
  * 添加原因：必填，最大允许输入 200 字符；占位符“请输入添加白名单的原因”
* 新增IP白名单：
  * IP信息输入：占位符展示为“请输入IP地址”；最大允许输入 100 字符；必填
  * 生效时间：必填，开始时间和结束时间均必填；占位符“请选择生效时间”和“请选择失效时间”；最大允许选择时长不超过 1 年；不允许选择过去时间
  * 添加原因：必填，最大允许输入 200 字符；占位符“请输入添加白名单的原因”
* 确认按钮：点击后触发校验，期间 loading；校验通过后展示二次确认弹窗
  * 填写完整性校验：若未填写完成，则对应未填写项输入框标红
  * 生效对象不存在：信息输入框标红，并 toast 提示“账号名不存在 / SDKID不存在 / 设备ID不存在 / IP不存在”
  * 信息重复性校验：若生效对象已存在，则 toast 提示“该账号已添加白名单 / 该设备已添加白名单 / 该IP已添加白名单”
* 二次确认弹窗：
  * 标题：提示
  * 描述：加入白名单的对象，在生效期间内不再触发风控。请确认是否添加
* 取消按钮&关闭按钮：点击后关闭，无二次确认；关闭后不保留上一次未提交的填写内容

2. **流程-编辑白名单**

* 弹窗类型：包括三种，同步页签选中状态，分为账号白名单、设备白名单、IP白名单

  * 配置项：仅允许配置时间，其他信息均为只读展示
  * 只读信息：以摘要方式展示 SDKID、账号名、添加原因
  * 生效时间：必填，开始时间和结束时间均必填；占位符“请选择生效时间”和“请选择失效时间”；最大允许选择时长不超过 1 年；不允许选择过去时间
* 编辑设备白名单：
  * 配置项：仅允许配置时间，其他信息均为只读展示
  * 只读信息：以摘要方式展示设备ID、添加原因
  * 生效时间：必填，开始时间和结束时间均必填；占位符“请选择生效时间”和“请选择失效时间”；最大允许选择时长不超过 1 年；不允许选择过去时间
* 编辑IP白名单：
  * 配置项：仅允许配置时间，其他信息均为只读展示
  * 只读信息：以摘要方式展示IP地址、添加原因
  * 生效时间：必填，开始时间和结束时间均必填；占位符“请选择生效时间”和“请选择失效时间”；最大允许选择时长不超过 1 年；不允许选择过去时间
* 二次确认弹窗：
  * 标题：提示
  * 描述：加入白名单的对象，在生效期间内不再触发风控。请确认是否更新生效时间
* 取消按钮&关闭按钮：点击后关闭，无二次确认；关闭后不保留上一次未提交的修改内容

* 交互效果：点击后拉起右侧抽屉
* 配置内容：包括提示区、导入操作区、解析结果列表
* 提示区：
  * 当前导入类型：账号/设备/IP 白名单导入，请上传对应模板文件，避免模板混用
  * 提示文案：
    * 上传模板：请先下载导入模板文件，按照格式填写后再上传。
    * 上传数量：每份文件最多支持上传 500 条数据。
    * 冲突校验：系统将根据 SDKID/设备ID/IP 进行冲突检测，重复条目导入后将覆盖更新生效时间。
    * 有效时长：白名单生效时长最大不超过 1 年，请在文件中明确截止日期。
  * 交互效果：点击“导入模板文件”触发浏览器下载
  * 模板文件：
    * 账号白名单：风控-账号白名单模板.xlsx
    * 设备白名单：风控-设备白名单模板.xlsx
    * IP白名单：风控-IP白名单模板.xlsx
  * 说明：模板下载按当前页签联动，只提供当前类型对应的模板文件
* 模板解析：
  * 状态展示：包括默认状态、解析中、解析完成状态、解析失败状态
  * 默认状态：展示文案“点击或将文件拖拽到这里上传 + 支持 .xls、.xlsx 格式文件”；点击后打开资源管理器选择上传文件
  * 解析中：展示“解析中...”
  * 解析完成：展示总解析数、解析成功数、解析失败数、覆盖更新数
  * 解析失败：展示失败态摘要和失败原因，不生成解析结果表，不允许提交导入
  * 下载解析结果：
    * 交互效果：点击后下载当前页签对应的解析结果示例文件
    * 文件表头：上传源文件右侧新增两列“解析结果”和“失败原因”
    * 失败原因：包括账号不存在/设备不存在/IP不存在、账号已添加过白名单/设备已添加过白名单/IP已添加过白名单
  * 重新上传：点击后打开资源管理器重新选择上传文件
  * 说明：解析结果下载按当前页签联动，只提供当前类型对应的解析结果文件
* 解析校验：
  * 模板类型校验：
    * 账号页签仅接受账号白名单模板
    * 设备页签仅接受设备白名单模板
    * IP页签仅接受IP白名单模板
    * 触发条件：上传的文件名类型与当前页签不匹配时触发
    * 失败提示：模板类型错误：当前为账号/设备/IP 白名单导入，请上传对应模板文件。
  * 文件格式校验：
    * 仅支持 `.xls`、`.xlsx`
    * 触发条件：上传非 `.xls/.xlsx` 文件时触发
    * 失败提示：文件格式错误：请上传 `.xls` 或 `.xlsx` 格式文件。
  * 完整性校验：未通过时提示文案如下
    * 请填写账号信息
    * 请填写设备ID
    * 请填写IP地址
    * 请填写生效时间
    * 请填写失效时间
    * 请填写添加原因
    * 触发条件：模板中对应字段为空时触发
  * 生效对象有效性校验：未通过时提示文案如下
    * 账号信息错误
    * 设备ID错误
    * IP地址错误
    * 触发条件：生效对象不存在或无法识别时触发
  * 生效时间有效性校验：未通过时提示文案如下
    * 时间格式错误
    * 生效时间已过期
    * 触发条件：时间格式不合法、起止时间缺失、生效时间已过期或时长超过 1 年时触发
  * 重复主键校验：
    * 账号按 SDKID，设备按设备ID，IP按IP地址识别重复
    * 触发条件：当前项目下已有同主键白名单记录时触发
    * 处理规则：不跳过，导入后覆盖更新已有白名单的生效时间和失效时间
* 解析结果：
  * 状态展示：包括默认状态（未解析）、解析中、解析完成状态、解析失败状态
  * 默认状态：展示文案“暂无解析结果，请先上传文件”
  * 解析中：展示文案“文件解析中，请稍候...”
  * 解析完成：展示列表和统计提示
    * 统计提示：共成功解析 X 条，其中 Y 条已添加过的白名单，导入后将覆盖更新生效时间
    * 列表表头-账号白名单：SDKID、账号名、生效时间、失效时间、添加原因
    * 列表表头-设备白名单：设备ID、生效时间、失效时间、添加原因
    * 列表表头-IP白名单：IP地址、生效时间、失效时间、添加原因
    * 展示规则：与页面内列表展示规则一致
    * 交互效果：支持水平拖动展示
  * 解析失败：展示失败态摘要，不展示可导入结果表
* 提交导入：
  * 交互效果：点击后展示二次确认弹窗
  * 提示文案：加入白名单的对象，在生效期间内不再触发风控。请确认是否添加
  * 导入结果：新增记录直接写入当前项目下的对应白名单；重复主键记录覆盖更新生效时间和失效时间
* 取消按钮&关闭按钮：
  * 空抽屉：点击后直接关闭
  * 已上传、解析中、解析完成、解析失败：点击“取消”或右上角关闭均触发二次确认
  * 确认文案：关闭后当前上传文件与解析结果将被清空，是否确认关闭？

**黑名单管理**

:::
主要调整：风控后台不再直接维护功能封禁能力，封禁能力统一收敛到封禁服务中。

原因如下：

登录、注册、支付等关键链路已有实时风险打分能力，高风险行为可在当次请求中实时拦截。  
实时风控适合处理即时行为风险，封禁管理适合处理确认后的长期对象限制，两者分层更合理。  
风控侧单独维护黑名单会与封禁管理重复建设，黑名单应统一归入封禁管理。  
统一封禁管理有利于查询、解除、审批、审计和跨业务系统接入。
:::
