# 多维画像查询页面方案

## 1. 文档定位

| 项目 | 内容 |
| --- | --- |
| 目标 | 形成“多维画像查询”首期可评审、可出图、可开发的正式页面方案 |
| 适用范围 | 用户画像中心 > 多维画像查询 |
| 当前阶段 | 需求收敛后的页面方案稿 |
| 方案基线 | 以当前多轮问询结论为准，旧方案仅作参考，不做结构约束 |

## 2. 页面目标

### 2.1 页面定位

“多维画像查询”不是纯资料展示页，也不是策略配置页，而是一个面向产品负责人、产品领导和游戏运营的**异常核实工作区**。

### 2.2 核心使用场景

| 场景 | 说明 |
| --- | --- |
| 盗号核实 | 运营收到玩家盗号反馈后，在后台快速核实近期登录异常、设备/IP 变化和风险命中情况 |
| 误封核实 | 产品/运营核实当前证据是否足以支撑异常判断，避免误解“没查到就是没问题” |
| 扩展场景 | 后续支持注册刷号、支付异常等其他风险场景 |

## 3. 页面容器方案

| 项目 | 方案 |
| --- | --- |
| 容器形态 | 大弹窗 |
| 原因 | 当前后台没有全屏组件规范，引入新组件得不偿失 |
| 复用方式 | 作为独立查询结果组件，可被其他页面快速调用 |
| 内部交互约束 | 不再叠加二次弹窗或抽屉，优先采用页内展开、切换、折叠 |

## 4. 查询对象与场景范围

### 5.1 查询入口

采用统一查询入口 + 查询类型选择，支持：

| 类型 | 查询字段 |
| --- | --- |
| 账号信息 | 账号名、SDKID |
| 设备信息 | 设备ID |
| 网络信息 | IP地址 |

### 5.2 首期场景范围

| 场景 | 首期状态 |
| --- | --- |
| 登录异常 | 首期主场景，优先完善 |
| 注册异常 | 首期纳入结构，便于后续防刷号扩展 |
| 支付异常 | 首期纳入结构，便于后续代充/支付风险扩展 |

## 5. 核心判断口径

### 6.1 结论边界

页面首期只先给出：

- `存在异常`
- `未发现明显异常`

不直接给出统一业务定性结论，例如：

- 更像盗号
- 更像误封
- 更像刷号
- 更像代充

原因：

1. 当前异常类型很多，不止盗号
2. 现有模型能力主要覆盖盗号风险打分
3. 误封、买卖号纠纷、代充等场景无法稳定自动定性

### 6.2 时间窗口

页面统一采用双时间窗：

| 时间窗 | 作用 |
| --- | --- |
| 近 7 天异常 | 判断最近是否出事 |
| 近 30 天历史基线 | 判断是否偏离常态 |

### 6.3 风险聚合方式

不做统一账号总风险分，账号视角的概览采用组合表达：

| 维度 | 展示方式 |
| --- | --- |
| 近期最高风险事件 | 展示近 7 天内最危险的一次事件 |
| 风险事件分布 | 展示异常次数/高风险次数/中风险次数等 |
| 最近风险时间 | 展示最近一次关键异常的发生时间 |

## 6. 完整上下结构

| 顺序 | 区域 | 默认状态 / 交互 | 作用 |
| --- | --- | --- | --- |
| 1 | 弹窗标题区 | 固定显示 | 标题带查询对象，右侧关闭按钮 |
| 2 | 基础信息区域 | 置顶保留，默认收起，一行三列展示 | 快速确认对象身份与当前状态 |
| 3 | 场景 Tab | 默认显示 | 固定顺序切换登录、注册、支付，并用红点或数字承接跨场景状态 |
| 4 | 登录分布模块 | 默认显示 | 用时间序列快速理解近 7 天风险事件分布，并在标题下承接一句风险摘要 |
| 5 | 关联分析区 | 默认显示 | 发现更深层问题，如设备复用、IP复用、账号聚集、工作室嫌疑 |
| 6 | 登录日志区 | 默认显示 | 提供风险记录 / 全量记录切换、表格明细和分页核查能力 |

## 7. 各模块设计说明

### 8.1 弹窗标题区

| 项目 | 说明 |
| --- | --- |
| 标题形式 | `多维画像查询 · 查询对象` |
| 查询对象示例 | 账号名 / SDKID / 设备ID / IP |
| 右侧操作 | 关闭按钮 |

### 8.2 基础信息区域

| 项目 | 说明 |
| --- | --- |
| 展示方式 | 默认收起的一行三列展示，置顶保留 |
| 目的 | 让用户始终确认“当前看的是谁”，并在需要时展开完整基础信息 |
| 字段原则 | 收起态展示最关键的三列字段，展开后进入完整基础信息区域 |

收起态字段：

| 查询对象 | 一行三列字段 |
| --- | --- |
| 查询账号 | SDKID、账号名、账号状态 |
| 查询设备 | 设备ID、设备型号、设备状态 |
| 查询IP | IP地址、IP归属地、IP状态 |

### 8.3 场景总览条

本轮已从结果弹窗主结构中移除场景总览条。

| 项目 | 说明 |
| --- | --- |
| 移除原因 | 默认高度下占据首屏空间，且在默认激活登录场景时价值下降 |
| 替代方式 | 跨场景状态由 `场景 Tab` 上的红点或数字承接 |
| 当前原则 | 结果弹窗进入后直接进入主核实链路，不再单独放全局概览条 |

### 8.4 场景 Tab

| 项目 | 说明 |
| --- | --- |
| 顺序 | 固定顺序：登录、注册、支付 |
| 风险表达 | 有风险的页签可使用红点或计数徽标 |
| 默认激活 | 为保持交互一致性，首版默认激活 `登录`，不替用户自动切页 |
| 设计原则 | 不采用动态重排，不在页签上堆叠“推荐查看”大标签 |

### 8.5 当前场景概览卡

本轮已从结果弹窗主结构中移除独立指标卡。

| 项目 | 说明 |
| --- | --- |
| 移除原因 | 与 `登录分布` 阅读路径脱节，单独占高 |
| 信息去向 | 原指标信息并入 `登录分布` 模块标题下的一句摘要 |
| 当前原则 | 不再用三张独立 KPI 卡片占据首屏高度 |

### 8.6 登录分布模块

| 项目 | 说明 |
| --- | --- |
| 作用 | 以时间序列快速说明近 7 天风险事件分布和关键风险节点 |
| 模块形态 | 独立白色卡片，位于场景 Tab 下方 |
| 模块头部 | 第一行显示模块标题 `登录分布`，第二行显示一行风险摘要 |
| 横轴 | 分两行展示：第一行月日，第二行时分秒 |
| 纵轴 | 风险分 |
| 节点规则 | 只对风险节点做特殊标记，保留悬浮信息 |
| 悬浮字段 | 时间、风险分、设备ID、IP地址 |

说明：

推荐摘要文案：

`近7日风险概览：风险事件 5 · 最高风险分 95 · 平均风险分 40`

说明：

1. 该模块是当前定稿中承接“异常经过”的主模块
2. 原三张指标卡的信息不丢失，只是并入模块头部
3. 首版不做复杂缩放分析器，不预设过多节点类型

### 8.7 关联分析区

关联信息不再仅仅视为“图谱展示”，而是**进一步发现问题的分析模块**，并且在当前定稿中位于 `登录分布` 与 `登录日志` 之间。

| 价值 | 示例 |
| --- | --- |
| 发现设备复用 | 一个设备登录大量账号，存在工作室嫌疑 |
| 发现 IP 复用 | 一个 IP 关联多个异常账号 |
| 发现账号聚集 | 同一批对象之间存在明显聚集关系 |

关联分析区内部结构采用两层：

1. 副标题展示关联统计信息
2. 下方展示关联图谱

交互规则：

| 交互 | 说明 |
| --- | --- |
| 节点移入 | 查看关联对象详情 |
| 节点点击 | 作为下钻入口预留，但首版前端可先不落复杂跳转 |

节点展示规则：

| 字段 | 说明 |
| --- | --- |
| 第一行 | 图标 + 标签名，例如 `常用设备`、`新增设备`、`常用IP`、`新增IP` |
| 第二行 | 对象唯一标识，例如 `DEV_412315`、`182.12.44.102` |
| 宽度规则 | 外围节点固定宽度，不因长标识撑宽 |
| 截断规则 | 标识过长时使用中间省略 |

### 8.8 登录日志区

| 项目 | 规则 |
| --- | --- |
| 模块标题 | 统一使用 `登录日志`，不再使用 `登录活动日志` |
| 模块定位 | 提供完整历史上下文与明细核查能力 |
| 默认视图 | 默认 `风险记录`，支持切到 `全部记录` |
| 默认排序 | 按时间倒序 |
| 列顺序 | `时间`、`风险分`、`设备`、`IP`、`结果`、`操作` |
| 时间列规则 | 时间前不加圆点、状态点或其他列表前缀 |
| 风险分规则 | `风险分` 固定放在第二列，靠近时间列 |
| 分页规则 | 使用分页器，默认每页 10 条 |
| 多页示例 | 设计基线使用 `共 37 条记录` 的多页示例；开发时可继续使用该 mock 口径 |
| 留白要求 | 保留正常表头高度、行高、分页器留白和底部统计区 |

不同查询对象的副标题文案：

| 查询对象 | 副标题文案 |
| --- | --- |
| 查询账号 | 账号 XXXXXXX 关联了 X 个设备，关联了 X 个 IP |
| 查询设备 | 设备 XXXXXXX 关联了 X 个账号，关联了 X 个 IP |
| 查询IP | IP XXXXXXX 关联了 X 个账号，关联了 X 个设备 |

## 8. 场景差异化原则

页面共享统一骨架，但当前定稿图只把“登录”场景细化到可开发粒度；`注册` 和 `支付` 先保留在 Tab 结构中，后续开发可沿用同一骨架补充各自内容。

| 场景 | 当前状态 | 说明 |
| --- | --- | --- |
| 登录 | 已定稿 | 已明确登录分布摘要、关联分析、登录日志的展示口径 |
| 注册 | 预留结构 | 先保留页签与风险状态表达，后续补场景专属内容 |
| 支付 | 预留结构 | 先保留页签与风险状态表达，后续补场景专属内容 |
| 支付异常 | 支付关系、支付账号复用、支付链路异常 |

## 10. 当前不建议纳入的方向

| 能力 | 不建议原因 |
| --- | --- |
| 全屏结果容器 | 当前后台无全屏组件规范，落地成本高 |
| Tab 动态重排 | 会造成结构跳变 |
| 场景自动业务定性 | 当前模型能力不足，容易误导 |
| 统一账号总风险分 | 单次事件分难以稳定聚合为可信总分 |
| 复杂时间轴缩放器 | 首版复杂度过高，容易偏离核实主线 |
| 弹窗内再开抽屉/弹窗 | 与大弹窗容器冲突，阅读路径会被打断 |

## 12. 首屏目标与摘要文案

### 12.1 首屏目标

1. 首屏至少看到 `登录分布`
2. 首屏至少看到 `关联分析`
3. 首屏至少看到 `登录日志` 标题或起始区

### 12.2 固定摘要文案

推荐在 `登录分布` 标题下使用单行横向短句：

```text
近7日风险概览：风险事件 5 · 最高风险分 95 · 平均风险分 40
```

### 12.3 使用原则

| 原则 | 说明 |
| --- | --- |
| 单行表达 | 不拆成二级卡片或徽标组 |
| 数字加粗 | 保持扫读性，但不再独立占模块高度 |
| 与图表同组 | 摘要句属于 `登录分布` 模块头部，不单独成模块 |

## 13. 当前最终口径快照

以下内容优先级最高，用于覆盖文档中任何早期过程性表述。

| 主题 | 当前最终口径 |
| --- | --- |
| 结果容器 | 大弹窗，可复用独立查询结果组件 |
| 顶部顺序 | 弹窗标题区 → 基础信息区域（默认收起）→ 场景 Tab |
| 基础信息区域 | 区域本身默认收起；收起态为一行三列，不同查询对象展示不同字段组合 |
| 场景总览条 | 本轮移除，不再作为当前结构基线 |
| 场景概览卡 | 本轮移除，指标信息并入 `登录分布` 模块头部 |
| 登录分布摘要 | `近7日风险概览：风险事件 5 · 最高风险分 95 · 平均风险分 40` |
| 平均风险分口径 | 仅基于近7天风险登录事件计算 |
| 时间线 | 横轴为时间，纵轴为风险分；只对风险节点做特殊标记 |
| 风险节点详情 | 时间、风险分、设备ID、IP地址 |
| 核实依据区 | 当前结果弹窗中不再单独展示 |
| 日志区 | 默认近7天风险记录，可切全量，并保留风险高亮 |
| 关联分析区 | 副标题统计信息 + 关联图谱 |
| 图谱节点悬浮 | 对象唯一标识 + 近7天风险登录次数 + 近7天最高风险分 + 近7日平均风险分 |
| 图谱节点点击 | 下钻分析，定位到对应对象 |
| 首屏目标 | 至少看到 `登录分布`、`关联分析`、`登录日志` 标题或起始区 |
| 首期范围 | P0 + P1 + 关联图谱 |

## 14. 基础信息区域字段定稿

### 14.1 布局规则

| 项目 | 结论 |
| --- | --- |
| 布局 | 一行三列 |
| 状态 | 置顶保留，默认收起 |
| 原则 | 基础信息区域本身采用收起态一行三列概览，展开后显示完整基础信息 |

### 14.2 不同查询对象字段

| 查询对象 | 一行三列字段 |
| --- | --- |
| 查询账号 | SDKID、账号名、账号状态 |
| 查询设备 | 设备ID、设备型号、设备状态 |
| 查询IP | IP地址、IP归属地、IP状态 |

### 14.3 设计说明

| 说明 | 原因 |
| --- | --- |
| 收起态只放三列 | 避免基础信息区域在默认态膨胀成资料区 |
| 展开后再看完整资料 | 兼顾快速识别和完整核查 |
| 标签信息不放收起态 | 标签更适合作为分析内容，不应抢置顶位置 |
| 最近活跃时间不放收起态 | 活跃时间更适合放在场景判断或日志上下文中 |

## 15. 风险节点详情字段定稿

### 15.1 时间线坐标定义

| 维度 | 定义 |
| --- | --- |
| 横轴 | 时间 |
| 纵轴 | 风险分 |

说明：时间线上的事件按时间分布展示，风险节点通过位置和样式共同体现风险高低。

### 15.2 风险节点内联展开字段

| 字段 | 用途 |
| --- | --- |
| 事件时间 | 确认风险事件发生时间 |
| 风险分 | 确认该次事件严重程度 |
| 登录账号 | 确认账号对象 |
| 设备信息 | 确认当次使用设备 |
| IP信息 | 确认当次使用IP |
| 具体得分项 | 说明该次风险分由哪些规则/项构成 |

### 15.3 设计原则

| 原则 | 说明 |
| --- | --- |
| 只展示解释风险成立所需最小字段 | 避免内联展开退化成第二张日志表 |
| 账号、设备、IP作为一组对象信息展示 | 方便快速复核本次事件环境 |
| 具体得分项必须可读 | 不仅展示总分，还要解释分从哪里来 |

## 16. 登录分布摘要字段定稿

| 字段 | 说明 |
| --- | --- |
| 近7天风险登录次数 | 作为异常强度摘要 |
| 近7天最高风险分 | 作为风险上限摘要 |
| 近7日平均风险分 | 作为近期整体风险水平辅助指标；仅基于近7天风险登录事件计算 |

### 16.1 定稿说明

| 说明 | 原因 |
| --- | --- |
| 不再单独放“是否存在明显登录异常” | 直接进入主核实链路，不再通过单独总览条承接 |
| 不再保留独立指标卡 | 独立 KPI 卡片单独占高，压缩了首屏主链路可见性 |
| 平均风险分只基于风险登录事件 | 若纳入全部登录事件，均值会被正常登录稀释，失去风险参考价值 |
| 不纳入新设备数 / 新IP数 | 新设备、新IP不天然等于风险，容易误导 |
| 指标信息并入分布模块头部 | 保留风险摘要，同时避免新增一层独立模块高度 |
| 设备/IP相关解释下放到关联分析或登录日志 | 优先让出首屏高度，保证主链路可见 |
